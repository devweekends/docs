## What You'll Master

### Module 1: SQL Foundations
Build a rock-solid foundation in SQL that will serve you throughout your career.
- SELECT queries, filtering, sorting, and pagination
- JOINs mastery (INNER, LEFT, RIGHT, FULL, CROSS, Self)
- Aggregations and GROUP BY with HAVING
- Subqueries and Common Table Expressions (CTEs)
- Window functions for analytics

### Module 2: Database Design & Modeling
Design databases that scale and maintain data integrity.
- Entity-Relationship modeling
- Normalization (1NF, 2NF, 3NF, BCNF) and when to denormalize
- Constraints, foreign keys, and referential integrity
- Schema design patterns for common applications
- Migration strategies for evolving schemas

### Module 3: Transactions & ACID
Build reliable applications that handle failures gracefully.
- ACID properties deep dive
- Transaction isolation levels explained
- Deadlocks, race conditions, and how to prevent them
- Optimistic vs pessimistic locking
- Distributed transactions and 2PC

### Module 4: Indexing Deep Dive
The single most important skill for database performance.
- B-Tree index internals and when to use them
- Hash, GIN, GiST, and BRIN indexes
- Composite indexes and column ordering
- Partial and expression indexes
- Index maintenance and bloat

### Module 5: Query Optimization
Turn slow queries into fast ones systematically.
- Reading EXPLAIN ANALYZE like a pro
- Query planner internals
- Common anti-patterns and fixes
- Join optimization strategies
- Query rewriting techniques

### Module 6: PostgreSQL Internals
Understand how the database actually works.
- Storage engine architecture
- MVCC and tuple visibility
- WAL and crash recovery
- Vacuum and autovacuum tuning
- Memory management and shared buffers

### Module 7: Replication & High Availability
Build systems that survive failures.
- Streaming replication setup
- Synchronous vs asynchronous replication
- Failover and switchover procedures
- Logical replication for migrations
- Connection pooling with PgBouncer

### Module 8: Scaling Strategies
Handle growth from thousands to millions of users.
- Read replicas and load balancing
- Sharding strategies and implementation
- Partitioning for large tables
- Caching layers (Redis integration)
- Multi-region deployment

### Module 9: Operational Excellence
Run databases in production with confidence.
- Monitoring and alerting setup
- Backup strategies and point-in-time recovery
- Performance tuning and configuration
- Security hardening
- Disaster recovery planning

## Who Is This For?

<Tabs>
  <Tab title="Backend Developers">
    You write application code that interacts with databases daily. Learn to:
    - Write efficient queries that don't slow down your app
    - Design schemas that evolve gracefully
    - Debug performance issues without calling the DBA
    - Make informed decisions about data modeling
  </Tab>
  
  <Tab title="Full-Stack Engineers">
    You build complete applications and need to understand the data layer:
    - Choose the right database for your use case
    - Implement proper data access patterns
    - Handle transactions correctly
    - Scale your database as users grow
  </Tab>
  
  <Tab title="Aspiring DBAs">
    You want to specialize in database administration:
    - Master PostgreSQL internals
    - Learn operational best practices
    - Understand replication and HA
    - Develop performance tuning expertise
  </Tab>
  
  <Tab title="Interview Prep">
    Database knowledge is crucial for technical interviews:
    - System design questions require database expertise
    - SQL problems are common in coding rounds
    - Understanding trade-offs impresses interviewers
    - Real-world experience stands out
  </Tab>
</Tabs>
