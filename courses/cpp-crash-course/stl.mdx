# STL (Standard Template Library)

The STL is a collection of ready-to-use classes and functions for data structures and algorithms. It saves you time and helps you write safer code.

---

## Containers
Containers store collections of data. Think of them as boxes for your stuff.

```cpp
#include <vector>
#include <map>
#include <set>

// Vector: dynamic array
std::vector<int> vec = {1, 2, 3};
vec.push_back(4);

// Map: key-value pairs
std::map<std::string, int> ages;
ages["John"] = 25;

// Set: unique values
std::set<int> numbers = {1, 2, 3};
```

---

## Algorithms
STL provides algorithms for searching, sorting, transforming, and more.

```cpp
#include <algorithm>
std::vector<int> nums = {5, 2, 8, 1, 9};

// Sort
std::sort(nums.begin(), nums.end());

// Find
auto it = std::find(nums.begin(), nums.end(), 8);

// Transform (apply a function to each element)
std::transform(nums.begin(), nums.end(), nums.begin(),
               [](int x) { return x * 2; });
```

---

## Iterators
Iterators are like pointers that let you move through containers.

```cpp
std::vector<int> vec = {1, 2, 3, 4, 5};
for (auto it = vec.begin(); it != vec.end(); ++it) {
    std::cout << *it << " ";
}
// Output: 1 2 3 4 5
```

---

## Best Practices
- Prefer STL containers over raw arrays.
- Use algorithms instead of manual loops when possible.
- Use `auto` for iterator types.

## Advanced: Custom Comparators

You can sort containers with custom logic using comparators.

```cpp
std::vector<std::pair<int, int>> v = {{1,2},{3,1},{2,4}};
std::sort(v.begin(), v.end(), [](auto& a, auto& b) {
    return a.second < b.second;
});
```

---

## Practical: Using std::unordered_map

`std::unordered_map` is a hash table for fast lookups.

```cpp
#include <unordered_map>
std::unordered_map<std::string, int> freq;
freq["apple"] = 2;
```
