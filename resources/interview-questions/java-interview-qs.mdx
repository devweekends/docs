# Java & Spring Boot Interview Questions (50+ Detailed Q&A)

## 1. Core Java Fundamentals

<AccordionGroup>
<Accordion title="1. JDK vs JRE vs JVM">
**Answer**:
*   **JVM**: Virtual Machine. Runs Bytecode.
*   **JRE**: JVM + Libraries (Runtime).
*   **JDK**: JRE + Tools (Compiler `javac`, Debugger).
</Accordion>

<Accordion title="2. OOP Concepts in Java">
**Answer**:
*   **Encapsulation**: Private fields, Getters/Setters.
*   **Inheritance**: `extends`. `super`.
*   **Polymorphism**: Overloading (Compile time), Overriding (Runtime).
*   **Abstraction**: Interfaces, Abstract Classes.
</Accordion>

<Accordion title="3. `String` vs `StringBuilder` vs `StringBuffer`">
**Answer**:
*   **String**: Immutable. String Pool. Slow concatenation.
*   **StringBuilder**: Mutable. Fast. Not Thread-safe.
*   **StringBuffer**: Mutable. Thread-safe (Synchronized). Slow.
</Accordion>

<Accordion title="4. `equals()` vs `==`">
**Answer**:
*   `==`: Reference comparison (Address).
*   `equals()`: Content comparison (Object). Must override `hashCode()` if overriding `equals()`.
</Accordion>

<Accordion title="5. Java Collections Framework">
**Answer**:
*   **List**: ArrayList (Dynamic Array), LinkedList.
*   **Set**: HashSet (Hash Table), TreeSet (Sorted).
*   **Map**: HashMap, TreeMap.
*   **Queue**: PriorityQueue.
</Accordion>

<Accordion title="6. `final` keyword">
**Answer**:
*   **Variable**: Constant.
*   **Method**: Cannot override.
*   **Class**: Cannot inherit (Immutable).
</Accordion>

<Accordion title="7. Exception Handling">
**Answer**:
*   **Checked**: Compiler enforces (IOException). Must catch or throw.
*   **Unchecked**: Runtime (NullPointer). Logic errors.
*   **Error**: System crash (OutOfMemory). Don't catch.
</Accordion>

<Accordion title="8. Interface vs Abstract Class">
**Answer**:
*   **Interface**: Multiple inheritance support. Contract. (Java 8+: Default methods).
*   **Abstract Class**: Shared state/behavior. Constructor allowed. Single inheritance.
</Accordion>

<Accordion title="9. Generics">
**Answer**:
Type safety compile time.
Type Erasure (Removed at runtime).
`List<T>`. Wildcards `? extends Number`.
</Accordion>

<Accordion title="10. Java 8 Features">
**Answer**:
Lambdas.
Streams API.
Optional.
Default Methods.
Functional Interfaces.
New Date Time API.
</Accordion>
</AccordionGroup>

## 2. JVM & Internals

<AccordionGroup>
<Accordion title="11. Memory Model (Stack vs Heap)">
**Answer**:
*   **Stack**: Treads. Local primitives, Method calls. LIFO. Auto-cleared.
*   **Heap**: Objects. Garbage Collected. Shared by threads.
</Accordion>

<Accordion title="12. Garbage Collection Roots">
**Answer**:
GC starts from Roots (Stack variables, Static variables, Active threads).
Mark and Sweep algorithm.
</Accordion>

<Accordion title="13. GC Algorithms">
**Answer**:
*   **Serial**: Single thread. Stop the world.
*   **Parallel**: Throughput focus.
*   **G1 (Garbage First)**: Region based. Low latency. Default modern.
*   **ZGC**: Ultra low latency.
</Accordion>

<Accordion title="14. ClassLoader Subsystem">
**Answer**:
Loading (`.class` -> Memory) -> Linking (Verify) -> Initialization.
Bootstrap -> Extension -> Application ClassLoader.
Delegation model.
</Accordion>

<Accordion title="15. Memory Leaks in Java">
**Answer**:
Unclosed Resources (Streams, DB).
Static Maps growing forever.
Inner classes holding reference to Outer.
ThreadLocal variables.
</Accordion>

<Accordion title="16. Just-In-Time (JIT) Compiler">
**Answer**:
Compiles hotspots of Bytecode to Native Machine Code during runtime for performance.
Inlining, Loop unrolling.
</Accordion>

<Accordion title="17. Comparison with C++">
**Answer**:
Java: Managed memory (GC), No pointers (References), Platform independent (Bytecode).
C++: Manual memory, Pointers, Compile to machine code.
</Accordion>

<Accordion title="18. Metaspace (Java 8+)">
**Answer**:
Replaced PermGen.
Stores Class Metadata.
Uses Native Memory (Not Heap). Auto-scales.
</Accordion>

<Accordion title="19. Reference Types">
**Answer**:
*   **Strong**: Normal.
*   **Weak**: Collected if only weak refs exist (WeakHashMap).
*   **Soft**: Collected if memory low (Cache).
*   **Phantom**: Post-mortem (Cleanup).
</Accordion>

<Accordion title="20. Volatile Keyword">
**Answer**:
Visibility Guarantee.
Prevents caching value in CPU register. Read/Write directly to RAM.
Happens-before relationship.
</Accordion>
</AccordionGroup>

## 3. Multithreading & Concurrency

<AccordionGroup>
<Accordion title="21. Thread Lifecycle">
**Answer**:
New -> Runnable -> Running -> Blocked/Waiting -> Terminated.
</Accordion>

<Accordion title="22. `Synchronized`">
**Answer**:
Intrinsic Lock (Monitor).
Block level or Method level.
Only one thread can access.
Performance cost.
</Accordion>

<Accordion title="23. Deadlock">
**Answer**:
Thread A holds Lock 1, wants Lock 2.
Thread B holds Lock 2, wants Lock 1.
Wait forever.
Fix: Lock Ordering. `tryLock`.
</Accordion>

<Accordion title="24. Executor Framework">
**Answer**:
Thread Pool management.
`FixedThreadPool`, `CachedThreadPool`.
Decouples submission from execution.
</Accordion>

<Accordion title="25. `Callable` vs `Runnable`">
**Answer**:
*   `Runnable`: `void run()`. No return, no exception.
*   `Callable`: `V call()`. Returns Future. Throws Exception.
</Accordion>

<Accordion title="26. `CompletableFuture`">
**Answer**:
Async programming.
Chaining (`thenApply`, `thenAccept`).
Combining (`allOf`).
Exception handling `exceptionally`.
</Accordion>

<Accordion title="27. Atomic Variables">
**Answer**:
`AtomicInteger`.
CAS (Compare And Swap) hardware instruction.
Lock-free thread safety.
</Accordion>

<Accordion title="28. Race Condition">
**Answer**:
Outcome depends on timing.
Read-Modify-Write is not atomic.
Fix: Synch or Atomic.
</Accordion>

<Accordion title="29. ThreadLocal">
**Answer**:
Variable local to the thread.
Transaction Context, User Session.
Danger: Memory leak in Pools (Must clean up).
</Accordion>

<Accordion title="30. Fork/Join Framework">
**Answer**:
Divide and Conquer (Recursion).
Work Stealing algorithm.
Used by Parallel Streams.
</Accordion>
</AccordionGroup>

## 4. Spring Boot & Framework

<AccordionGroup>
<Accordion title="31. Dependency Injection (IoC)">
**Answer**:
Inversion of Control.
Container creates and manages Objects (Beans) and injects dependencies using `@Autowired`.
Decoupling.
</Accordion>

<Accordion title="32. Bean Lifecycle">
**Answer**:
Instantiate -> Populate Properties -> `BeanNameAware` -> `PostProcessBeforeInit` -> `@PostConstruct` -> `PostProcessAfterInit` -> Ready.
</Accordion>

<Accordion title="33. Spring Scopes">
**Answer**:
*   **Singleton** (Default): One per container.
*   **Prototype**: New every time.
*   **Request**: Per HTTP req.
*   **Session**: Per User session.
</Accordion>

<Accordion title="34. `@SpringBootApplication`">
**Answer**:
Combines `@Configuration`, `@EnableAutoConfiguration`, `@ComponentScan`.
Magic starter.
</Accordion>

<Accordion title="35. Spring Boot Starter">
**Answer**:
Opinionated dependency bundle.
`spring-boot-starter-web` includes Tomcat, Jackson, Spring MVC.
Auto-configures classpath beans.
</Accordion>

<Accordion title="36. PathVariable vs RequestParam">
**Answer**:
*   `@PathVariable`: `/users/1` (RESTful Resource ID).
*   `@RequestParam`: `/users?id=1` (Filtering/Sorting).
</Accordion>

<Accordion title="37. Spring Data JPA">
**Answer**:
Abstraction over Hibernate.
Repository Interface (`extends JpaRepository`).
Derived queries (`findByEmail`).
</Accordion>

<Accordion title="38. Transaction Management (`@Transactional`)">
**Answer**:
Declarative.
AOP proxy.
Begins transaction -> Runs method -> Commit (or Rollback on RuntimeException).
Gotcha: Self-invocation bypasses proxy.
</Accordion>

<Accordion title="39. Spring AOP (Aspect Oriented Programming)">
**Answer**:
Cross-cutting concerns (Logging, Security, Tx).
Aspect, Advice (Before, After), Pointcut (Where).
</Accordion>

<Accordion title="40. Profiles">
**Answer**:
Environment specific config.
`application-dev.properties`, `application-prod.properties`.
`@Profile("dev")`.
</Accordion>
</AccordionGroup>

## 5. Microservices & Best Practices

<AccordionGroup>
<Accordion title="41. RestTemplate vs WebClient">
**Answer**:
*   **RestTemplate**: Blocking (Sync). Deprecated.
*   **WebClient**: Non-blocking (Async/Reactive). Modern.
</Accordion>

<Accordion title="42. Spring Boot Actuator">
**Answer**:
Production ready features.
Endpoints: `/health`, `/metrics`, `/info`.
Monitoring exposure.
</Accordion>

<Accordion title="43. Annotation vs XML Config">
**Answer**:
Legacy Spring used XML.
Modern Spring uses Annotations (Java Config). Type safe. Refactorable.
</Accordion>

<Accordion title="44. Handling Circular Dependencies">
**Answer**:
Bean A needs B, B needs A.
Error.
Fix: `@Lazy`, Setter injection, Refactor design.
</Accordion>

<Accordion title="45. Hibernate N+1 Problem">
**Answer**:
Fetching List (1 query), then Loop fetching Child for each (N queries).
Fix: `JOIN FETCH` (Eager loading in query).
</Accordion>

<Accordion title="46. DTO Pattern">
**Answer**:
Data Transfer Object.
Decouple Entity (DB) from API (Network).
Prevents over-fetching. Hide password fields.
MapStruct/ModelMapper.
</Accordion>

<Accordion title="47. Testing in Spring Boot">
**Answer**:
*   `@SpringBootTest`: Loads full context. Slow.
*   `@WebMvcTest`: Slice test for Controller. Mock Service.
*   `@DataJpaTest`: Slice for Repo. H2 DB.
</Accordion>

<Accordion title="48. Spring Security">
**Answer**:
Filter Chain.
Authentication (Who) & Authorization (Access).
OAuth2 / JWT filters.
</Accordion>

<Accordion title="49. Reactive Programming (Spring WebFlux)">
**Answer**:
Project Reactor (Mono, Flux).
Async, Event-driven, Non-blocking I/O.
Backend pressure handling.
</Accordion>

<Accordion title="50. Logging Best Practices (SLF4J)">
**Answer**:
Use Interface SLF4J, Implementation Logback.
`log.info("User {}", id)`.
Async Appender for performance.
</Accordion>
</AccordionGroup>
