# JavaScript Interview Questions (50+ Detailed Q&A)

## 1. Core Engine & Execution Context

<AccordionGroup>
<Accordion title="1. Explain Execution Context and the Call Stack">
**Answer**:
The **Execution Context (EC)** is the environment where JavaScript code runs. It contains:
1.  **Variable Environment**: `var`, `function` declarations.
2.  **Scope Chain**: Reference to outer environments.
3.  **`this` binding**.

Types:
*   **Global EC**: Created on startup.
*   **Function EC**: Created when a function is *invoked*.

The **Call Stack** is a LIFO stack. When a function executes, its EC is pushed. When it returns, it's popped.

**Follow-up**: What happens if the stack overflows? (MaxRecursionError). How does `this` change in an arrow function vs regular? (Arrow inherits from parent scope).
</Accordion>

<Accordion title="2. Hoisting Internals: var vs let/const">
**Answer**:
Hoisting moves declarations to the top of the scope used during the **Creation Phase**.
*   `var`: Hoisted and initialized with `undefined`. Accessing before declaration returns `undefined`.
*   `let/const`: Hoisted but NOT initialized. They enter the **Temporal Dead Zone (TDZ)**. Accessing them throws `ReferenceError`.
*   `function`: Fully hoisted. Can be called before definition.

**Follow-up**: What is the TDZ? (The lines of code between the scope start and the let/const declaration).
</Accordion>

<Accordion title="3. Event Loop: Microtasks vs Macrotasks">
**Answer**:
JS is single-threaded. The Event Loop manages async ops.
1.  **Call Stack**: Sync code.
2.  **Microtask Queue**: `Promise.then`, `process.nextTick`, `MutationObserver`. **High Priority**.
3.  **Macrotask Queue**: `setTimeout`, `setInterval`, I/O. **Low Priority**.
**Rule**: The loop clears the *entire* Microtask queue after every Macrotask.

**Follow-up**: Predict output (Promise vs Timeout). Can a Microtask block the UI? (Yes, infinite microtask loop hangs the browser).
</Accordion>

<Accordion title="4. Scopes & Scope Chain">
**Answer**:
Scope determines accessibility.
*   **Lexical Scope**: Scope is determined by *where code is written*, not where called.
*   **Scope Chain**: If a var isn't found in current scope, JS looks up to the parent scope, until Global.

**Follow-up**: What is dynamic scoping? (JS doesn't use it, but `this` behaves somewhat dynamically).
</Accordion>

<Accordion title="5. Closures: What, Why, and Memory Leaks">
**Answer**:
A closure is a function bundled with its lexical environment. It allows a function to access variables from an outer function *after* the outer function has returned.
**Use Cases**: Data privacy (Module pattern), Currying, Memoization.
**Memory**: Closures keep variables alive. If attached to a global or DOM element and not cleared, they cause leaks.

**Follow-up**: Implement a private counter.
```javascript
const counter = () => { let c=0; return () => ++c; }
```
</Accordion>

<Accordion title="6. `this` Keyword binding rules">
**Answer**:
1.  **Default**: `window` (strict: `undefined`).
2.  **Implicit**: `obj.func()` -> `this` is `obj`.
3.  **Explicit**: `.call(ctx)`, `.apply(ctx)`, `.bind(ctx)`.
4.  **New**: `new Func()` -> `this` is the new object.
5.  **Arrow**: Inherits `this` from surrounding scope (Lexical).

**Follow-up**: How to fix `this` in a class callback? (Bind in constructor or use Arrow function).
</Accordion>

<Accordion title="7. Call, Apply, Bind Polyfills">
**Answer**:
*   `call(thisArg, a, b)`: Invokes fn immediately.
*   `apply(thisArg, [args])`: Invokes fn immediately with array.
*   `bind(thisArg, a, b)`: Returns a *new function* with fixed `this`.

**Follow-up**: Implement `Function.prototype.myBind`.
</Accordion>

<Accordion title="8. Garbage Collection Algorithms">
**Answer**:
*   **Mark and Sweep**: Starts from Roots (Window, Stack). Marks reachable objects. Sweeps (deletes) unreachable ones.
*   **Generational**: Young generation (short lived) vs Old generation.
*   **Reference Counting**: Old method. Fails on circular references (`a.ref = b; b.ref = a`).

**Follow-up**: How to debug memory leaks? (Chrome DevTools Heap Snapshot).
</Accordion>

<Accordion title="9. Strict Mode: What does it do?">
**Answer**:
`'use strict';`
1.  Prevents accidentally creating global vars (`x = 5` throws error).
2.  Disables `with` statement.
3.  `this` in specific functions becomes `undefined` instead of `window`.
4.  Catches duplicate parameter names.
</Accordion>

<Accordion title="10. Memory Life Cycle">
**Answer**:
1.  **Allocation**: OS allocates memory (vars, objs).
2.  **Use**: Read/Write.
3.  **Release**: GC frees it when no longer needed.
</Accordion>
</AccordionGroup>

## 2. Types & Coercion

<AccordionGroup>
<Accordion title="11. Primitive vs Reference Types">
**Answer**:
*   **Primitives**: String, Number, Boolean, Null, Undefined, Symbol, BigInt. Stored in **Stack**. Immutable. Passed by value.
*   **Reference**: Objects, Arrays, Functions. Stored in **Heap**. Variable holds a pointer. Passed by reference.

**Follow-up**: `a = []; b = a; b.push(1);` Does `a` change? (Yes).
</Accordion>

<Accordion title="12. Coercion: Explicit vs Implicit">
**Answer**:
*   **Implicit**: `1 + '2' = '12'`, `5 - '2' = 3`.
*   **Falsy values**: `false`, `0`, `""`, `null`, `undefined`, `NaN`.
*   **Tricky**: `[] == ![]` (True). `null == undefined` (True).

**Follow-up**: Why is `[] + {}` -> `"[object Object]"`?
</Accordion>

<Accordion title="13. `typeof` vs `instanceof`">
**Answer**:
*   `typeof`: Returns string. Good for primitives. Bug: `typeof null === 'object'`.
*   `instanceof`: Checks prototype chain. `arr instanceof Array` (True).
*   **Best Check**: `Object.prototype.toString.call(val)` -> `[object Array]`.

**Follow-up**: How to check for Array reliably? (`Array.isArray()`).
</Accordion>

<Accordion title="14. Symbol Type">
**Answer**:
Unique, immutable primitive. Used for object keys to avoid collision. Hidden from `for...in`.
```javascript
const sym = Symbol('foo');
```
**Follow-up**: What is `Symbol.iterator`? (Makes an object iterable in `for...of`).
</Accordion>

<Accordion title="15. `null` vs `undefined`">
**Answer**:
*   `undefined`: Variable declared but not assigned. Default return value. "System error".
*   `null`: Assignment value. "Programmatic empty". Object wrapper.

**Follow-up**: `null + 1`? (1). `undefined + 1`? (NaN).
</Accordion>

<Accordion title="16. NaN (Not a Number)">
**Answer**:
"Invalid Number".
`typeof NaN === 'number'`.
`NaN === NaN` is **FALSE**.
**Check**: `Number.isNaN(val)`.

**Follow-up**: Difference between global `isNaN()` and `Number.isNaN()`? (Global coercing non-numbers to numbers first. `Number.isNaN` is strict).
</Accordion>

<Accordion title="17. BigInt">
**Answer**:
For integers larger than `2^53 - 1` (MAX_SAFE_INTEGER). Created via `10n` or `BigInt(10)`. Cannot mix with Numbers in math.
</Accordion>

<Accordion title="18. Wrapper Objects">
**Answer**:
Primitives like string have methods (`.toUpperCase()`) because JS temporarily wraps them in a transient Object (`new String()`) then discards it.
</Accordion>

<Accordion title="19. Mutable vs Immutable Patterns">
**Answer**:
*   **Mutable**: `arr.push()`, `obj.prop = 1`.
*   **Immutable**: `[...arr, 1]`, `{...obj, prop: 1}`. React requires immutability to detect changes.
</Accordion>

<Accordion title="20. Deep Copy vs Shallow Copy">
**Answer**:
*   **Shallow**: Copied top level. Nested objects are referenced. (`Object.assign`, spread).
*   **Deep**: Fully recursive copy.
    *   `JSON.parse(JSON.stringify(obj))` (Fails on Dates, Functions, undefined).
    *   `structuredClone(obj)` (Modern, Native).
    *   Lodash `_.cloneDeep`.
</Accordion>
</AccordionGroup>

## 3. Async JavaScript

<AccordionGroup>
<Accordion title="21. Callback Hell & Inversion of Control">
**Answer**:
Nested callbacks make code unreadable. Inversion of Control: You give control of your callback to a third party (e.g., analytics lib) which might call it 0 times, 1 time, or 100 times. Promises solve this by returning control (you decide what to do with the result).
</Accordion>

<Accordion title="22. Promises Steps & States">
**Answer**:
States: `Pending`, `Fulfilled`, `Rejected`.
*   `new Promise()` executor runs immediately.
*   `.then()` returns a NEW promise.
*   `.catch()` handles errors.

**Follow-up**: How to cancel a promise? (AbortController).
</Accordion>

<Accordion title="23. Async/Await Transpilation">
**Answer**:
Syntactic sugar over Promises/Generators.
Babel converts it to a `switch` case inside a generator runner. It pauses execution at `yield` (await) and resumes when promise resolves.
</Accordion>

<Accordion title="24. `Promise.all` vs `allSettled` vs `race` vs `any`">
**Answer**:
*   `all`: Fails fast. If one rejects, all reject.
*   `allSettled`: Waits for all. Returns array of `{status, value/reason}`.
*   `race`: First one to settle (resolve OR reject).
*   `any`: First one to **resolve**. Ignores rejections unless all reject.
</Accordion>

<Accordion title="25. Error Handling in Async">
**Answer**:
*   Callbacks: `if (err) ...`
*   Promise: `.catch()`.
*   Async/Await: `try { ... } catch (e) { ... }`.
*   **Uncaught**: `window.onunhandledrejection`.
</Accordion>

<Accordion title="26. Throttling and Debouncing Implementation">
**Answer**:
*   **Debounce**: "Wait until 500ms passed with no events." (Autocomplete).
*   **Throttle**: "Fire at most once every 500ms." (Scroll listeners).
**(Code required for interview)**.
</Accordion>

<Accordion title="27. Generators and Iterators">
**Answer**:
*   **Iterator**: Object with `next()` returning `{value, done}`.
*   **Generator**: `function*`. Returns an iterator. Can `yield` values and *pause* execution. Bi-directional communication (`next(val)` passes value back).
</Accordion>

<Accordion title="28. `setTimeout` vs `setImmediate` vs `process.nextTick`">
**Answer**:
(Node context)
*   `nextTick`: Immediate, before IO.
*   `setTimeout`: Timer phase.
*   `setImmediate`: Check phase (After IO).
</Accordion>

<Accordion title="29. Web Workers">
**Answer**:
Run scripts in background threads. Separate global scope/EC. No DOM access. Communicate via `postMessage`. Used for heavy computation (Image processing) to avoid freezing UI.
</Accordion>

<Accordion title="30. AJAX & Fetch API">
**Answer**:
*   **Old**: `XMLHttpRequest` (Events based).
*   **New**: `fetch` (Promise based). Note: `fetch` doesn't reject on 404/500, only on network error. You must check `res.ok`.
</Accordion>
</AccordionGroup>

## 4. DOM & Browser

<AccordionGroup>
<Accordion title="31. Event Propagation (Bubbling vs Capturing)">
**Answer**:
1.  **Capture Phase**: Root -> Target.
2.  **Target Phase**: At element.
3.  **Bubble Phase**: Target -> Root.
**Stop**: `e.stopPropagation()`.
**Delegation**: Putting listener on Parent to capture children events (`e.target`).
</Accordion>

<Accordion title="32. DOM Repaint vs Reflow">
**Answer**:
*   **Reflow (Layout)**: Calculating geometry (Width, Height, Position). Expensive.
*   **Repaint**: Changing look without geometry (Color, Visibility). Cheaper.
*   **Force Reflow**: Reading `offsetWidth` immediately after writing style.
</Accordion>

<Accordion title="33. LocalStorage vs SessionStorage vs Cookie">
**Answer**:
*   **Local**: Permanent, 5MB, Client only.
*   **Session**: Tab life, 5MB, Client only.
*   **Cookie**: Sent with requests, 4KB, Server readable. (HttpOnly, Secure flags).
</Accordion>

<Accordion title="34. Critical Rendering Path">
**Answer**:
HTML -> DOM
CSS -> CSSOM
DOM + CSSOM -> Render Tree -> Layout -> Paint -> Composite.
**Optimization**: Async scripts, Critical CSS.
</Accordion>

<Accordion title="35. Service Workers">
**Answer**:
Script running in background. Network proxy.
**Uses**: Offline support (Cache API), Push Notifications, Background Sync.
</Accordion>

<Accordion title="36. Shadow DOM">
**Answer**:
Encapsulated DOM tree. CSS inside doesn't leak out. Base of **Web Components**.
</Accordion>

<Accordion title="37. CORS (Cross-Origin Resource Sharing)">
**Answer**:
Browser security blocking requests to different domain.
Server must send `Access-Control-Allow-Origin` header.
**Preflight**: `OPTIONS` request sent first for non-simple requests (Custom headers, PUT/DELETE).
</Accordion>

<Accordion title="38. Browser Cache Mechanisms">
**Answer**:
*   **Memory Cache**: RAM (Current Session).
*   **Disk Cache**: Persistent.
*   **Cache-Control Header**: `max-age`, `no-cache`, `no-store`.
*   **ETag**: Version hash. 304 Not Modified.
</Accordion>

<Accordion title="39. `defer` vs `async` scripts">
**Answer**:
*   **Default**: Parses HTML -> Stops -> Download JS -> Exec JS -> Continues HTML.
*   **Async**: Downloads parallel. Executes ASAP (pauses HTML). Order not guaranteed.
*   **Defer**: Downloads parallel. Executes *after* HTML parsing. Order preserved. **(Best Practice)**.
</Accordion>

<Accordion title="40. BOM vs DOM">
**Answer**:
*   **DOM**: `document` (The page content).
*   **BOM**: `window` (The browser window: `location`, `history`, `navigator`, `screen`).
</Accordion>
</AccordionGroup>

## 5. ES6+ & Modern JS

<AccordionGroup>
<Accordion title="41. Map/Set vs Object/Array">
**Answer**:
*   **Map**: Keys can be anything (Objects), Preserves order, Size property, Iterable.
*   **Set**: Unique values only.
*   **WeakMap/WeakSet**: Keys must be objects. Held weakly (GC can delete if no other Ref). Good for metadata.
</Accordion>

<Accordion title="42. Destructuring & Spread">
**Answer**:
`const {a, b} = obj`. `const [x, y] = arr`.
**Spread**: `[...arr]`. Shallow copy.
**Rest**: `function(...args)`. Collects arguments into array.
</Accordion>

<Accordion title="43. Modules (ESM vs CJS)">
**Answer**:
*   **CJS (`require`)**: Dynamic, Sync, Node default.
*   **ESM (`import`)**: Static (analyzed at compile time/parse time), Async aware, Tree-shakeable.
</Accordion>

<Accordion title="44. Arrow Functions Differences">
**Answer**:
1.  No own `this`.
2.  No `arguments` object.
3.  Cannot be used as Constructor (`new`).
4.  Implicit return for single expression.
</Accordion>

<Accordion title="45. Proxies & Reflect">
**Answer**:
*   **Proxy**: Intercept basic operations (get, set, delete) on objects. Used in Vue 3 reactivity.
*   **Reflect**: Standard API to forward operations to original object.
</Accordion>

<Accordion title="46. Optional Chaining & Nullish Coalescing">
**Answer**:
*   `?.`: Safe access `obj?.prop`. Returns undefined if null/undefined.
*   `??`: `val ?? default`. Returns default ONLY if val is `null` or `undefined`. (Unlike `||` which triggers on 0 or "").
</Accordion>

<Accordion title="47. Iterators & Generators Uses">
**Answer**:
Redux Saga, Custom Async flows, infinite streams.
</Accordion>

<Accordion title="48. Classes vs Prototypes">
**Answer**:
Classes are syntax sugar.
`class` methods are non-enumerable. `class` uses strict mode. `class` constructor must be called with `new`.
</Accordion>

<Accordion title="49. Tagged Template Literals">
**Answer**:
`tag`string``.
Function `tag(strings, ...values)` receives parts.
Used in: `styled-components`, `graphql` queries to parse string.
</Accordion>

<Accordion title="50. Currying & Partial Application">
**Answer**:
*   **Currying**: `f(a,b,c)` -> `f(a)(b)(c)`.
*   **Partial**: Fixing some arguments. `const add5 = add.bind(null, 5)`.
Useful for functional composition.
</Accordion>
</AccordionGroup>
