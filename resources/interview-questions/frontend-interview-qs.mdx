# Frontend Engineer Interview Questions (50+ Detailed Q&A)

## 1. Browser Internals & Network

<AccordionGroup>
<Accordion title="1. What happens when you type a URL?">
**Answer**:
1.  DNS Lookup.
2.  TCP Handshake (SYN-ACK).
3.  SSL/TLS Handshake.
4.  HTTP GET Request.
5.  Server Response (HTML).
6.  Parsing (DOM, CSSOM).
7.  Render Tree -> Layout -> Paint -> Composite.
</Accordion>

<Accordion title="2. Critical Rendering Path">
**Answer**:
Sequence to render page.
Optimize: Minimize blocking CSS/JS.
Inline critical CSS. Async/Defer scripts.
</Accordion>

<Accordion title="3. Event Bubbling vs Capturing">
**Answer**:
*   **Capture**: Root -> Target (Down).
*   **Bubble**: Target -> Root (Up). (Default).
*   `stopPropagation()` stops bubbling.
*   **Delegation**: Attach listener to Parent to handle dynamic Children using bubbling.
</Accordion>

<Accordion title="4. Reflow vs Repaint">
**Answer**:
*   **Reflow (Layout)**: Geometry changes (Width, Position). Expensive.
*   **Repaint**: Appearance changes (Color, Visibility). Cheaper.
*   **Composite**: GPU Only (Transform, Opacity). Best performance.
</Accordion>

<Accordion title="5. HTTP/2 and HTTP/3">
**Answer**:
*   **HTTP/2**: Multiplexing, Header compression, Server Push.
*   **HTTP/3**: QUIC (UDP). No Head-of-Line blocking. Faster handshake.
</Accordion>

<Accordion title="6. Browser Cache Strategies">
**Answer**:
*   `Cache-Control`: max-age, no-cache, no-store.
*   `ETag`: Validation token. 304 Not Modified.
*   Service Worker Cache (Programmatic).
</Accordion>

<Accordion title="7. Worker Threads">
**Answer**:
*   **Web Worker**: Background CPU task. No DOM access.
*   **Service Worker**: Network Proxy. PWA. Offline.
*   **Worklet**: Rendering hook (Paint API).
</Accordion>

<Accordion title="8. V8 Garbage Collection">
**Answer**:
Generational (Nursery, Old Gen).
Incremental marking.
Orinoco (Parallel GC).
</Accordion>

<Accordion title="9. Resource Hints">
**Answer**:
`preload` (High priority, needed soon).
`prefetch` (Low priority, next nav).
`preconnect` (DNS+TCP ahead).
</Accordion>

<Accordion title="10. CORS (Cross Origin Resource Sharing)">
**Answer**:
Security boundary.
Browser blocks request to different domain unless Server sends `Access-Control-Allow-Origin`.
Preflight (OPTIONS).
</Accordion>
</AccordionGroup>

## 2. CSS Mastery

<AccordionGroup>
<Accordion title="11. Box Model">
**Answer**:
`width` + `padding` + `border` + `margin`.
`box-sizing: border-box` includes padding/border in width. (Standard).
</Accordion>

<Accordion title="12. Flexbox vs Grid">
**Answer**:
*   **Flex**: 1D (Row OR Column). Content-first.
*   **Grid**: 2D (Row AND Column). Layout-first.
</Accordion>

<Accordion title="13. Specificity Calculation">
**Answer**:
(Inline, ID, Class, Element).
`!important` overrides (Bad practice).
`0-1-0-0` (ID) beats `0-0-1-0` (Class).
</Accordion>

<Accordion title="14. BEM Methodology">
**Answer**:
Block Element Modifier.
`.card__title--large`.
Prevents bleeding styles. Modular.
</Accordion>

<Accordion title="15. Centering strategies">
**Answer**:
*   Flex: `justify-content: center; align-items: center`.
*   Grid: `place-items: center`.
*   Absolute: `top: 50%; left: 50%; translate: -50% -50%`.
</Accordion>

<Accordion title="16. Responsive Design">
**Answer**:
Media Queries based on viewport width.
Fluid typography (`clamp`).
Relative units (rem/em/vh/vw).
</Accordion>

<Accordion title="17. Stacking Context (z-index)">
**Answer**:
`z-index` only works if container creates context (Position relative, Opacity < 1, Transform, Flex child).
New context resets stack.
</Accordion>

<Accordion title="18. Pseudo-classes vs Pseudo-elements">
**Answer**:
*   **Class**: State (`:hover`, `:nth-child`).
*   **Element**: Part (`::before`, `::after`).
</Accordion>

<Accordion title="19. CSS Variables">
**Answer**:
`--main-color: red`.
Dynamic (JS can update). Scoped to DOM.
Preprocessor variables (Sass) are static compile time.
</Accordion>

<Accordion title="20. Containing Block">
**Answer**:
Reference box for `position: absolute` and `%` sizes.
Usually nearest ancestor with Position != Static.
</Accordion>
</AccordionGroup>

## 3. Web Performance (Core Web Vitals)

<AccordionGroup>
<Accordion title="21. LCP (Largest Contentful Paint)">
**Answer**:
Load speed of main content (Hero image).
Goal: < 2.5s.
Fix: Optimize image, Preload, remove render blocking JS.
</Accordion>

<Accordion title="22. INP (Interaction to Next Paint)">
**Answer**:
Responsiveness. Input latency.
Replaced FID.
Goal: < 200ms.
Fix: Break long tasks, yield to main thread.
</Accordion>

<Accordion title="23. CLS (Cumulative Layout Shift)">
**Answer**:
Visual Stability. Janky movement.
Goal: < 0.1.
Fix: Size attributes on images, reserve space for ads.
</Accordion>

<Accordion title="24. Image Optimization">
**Answer**:
Next-gen formats (WebP/AVIF).
Responsive (`srcset`).
Lazy Loading (`loading="lazy"`).
Compression.
</Accordion>

<Accordion title="25. Code Splitting">
**Answer**:
Break JS bundle into chunks.
Route-based or Component-based.
Webpack/Vite handles this.
</Accordion>

<Accordion title="26. Tree Shaking">
**Answer**:
Removing unused code (Dead code elimination).
Relies on ES Modules (static analysis of imports).
</Accordion>

<Accordion title="27. CDN Usage">
**Answer**:
Edge caching static assets.
Lower latency.
Geo-distribution.
</Accordion>

<Accordion title="28. Minification & Gzip/Brotli">
**Answer**:
Remove whitespace/comments. Rename variables short.
Compression: Brotli > Gzip.
</Accordion>

<Accordion title="29. Rendering Patterns (SSR vs CSR vs SSG)">
**Answer**:
*   **CSR**: Shell -> API -> Render. SEO harder.
*   **SSR**: HTML on Server. Good SEO. TTFB slower.
*   **SSG**: Static HTML at Build. Fastest.
*   **ISR**: Revalidate Static occasionally.
</Accordion>

<Accordion title="30. Memory Leaks">
**Answer**:
Detached DOM nodes.
Timers/Listeners not cleared.
Closure references.
</Accordion>
</AccordionGroup>

## 4. Accessibility (A11y)

<AccordionGroup>
<Accordion title="31. Semantic HTML">
**Answer**:
Use `<button>`, `<nav>`, `<main>`, `<article>` instead of `<div>`.
Screen reader understands structure.
Keyboard navigation works default.
</Accordion>

<Accordion title="32. ARIA Roles/Attributes">
**Answer**:
Accessible Rich Internet Applications.
Bridge gap where HTML fails.
`aria-label`, `aria-expanded`, `role="dialog"`.
Rule 1: Don't use ARIA if HTML element exists.
</Accordion>

<Accordion title="33. Keyboard Navigation">
**Answer**:
Everything clickable must be focusable (Tab index).
Visible Focus Ring (`outline`).
Enter/Space to activate.
</Accordion>

<Accordion title="34. Alt Text">
**Answer**:
Descriptive text for images.
SEO + Screen Readers.
Decorative images: `alt=""`.
</Accordion>

<Accordion title="35. Contrast Ratios">
**Answer**:
WCAG AA: 4.5:1 for normal text.
Tools: Lighthouse, WAVE.
</Accordion>

<Accordion title="36. Testing A11y">
**Answer**:
Automated: Axe/Lighthouse (Catch 30%).
Manual: Keyboard only. Screen Reader (NVDA/VoiceOver).
</Accordion>

<Accordion title="37. Reduced Motion">
**Answer**:
`@media (prefers-reduced-motion)`.
Disable large animations for vestibular disorders.
</Accordion>

<Accordion title="38. Skip Links">
**Answer**:
Link at top "Skip to Content".
Allows keyboard users to bypass Nav Menu.
</Accordion>

<Accordion title="39. Headings Hierarchy">
**Answer**:
`h1` -> `h2` -> `h3`. No skipping levels.
Navigable outline.
</Accordion>

<Accordion title="40. Form Labels">
**Answer**:
Always link `<label for="id">` to Input.
Or wrap Input in Label.
Placeholders are not labels.
</Accordion>
</AccordionGroup>

## 5. Security & Modern Web

<AccordionGroup>
<Accordion title="41. XSS (Cross Site Scripting)">
**Answer**:
Attacker injects script.
*   **Reflected**: Via URL.
*   **Stored**: In DB (Comment).
*   **Fix**: Content Security Policy (CSP), Escape Output, HttpOnly Cookies.
</Accordion>

<Accordion title="42. CSRF (Cross Site Request Forgery)">
**Answer**:
Attacker tricks user to call API (Bank transfer) using their cookie.
**Fix**: SameSite Cookie (Strict/Lax), Anti-CSRF Token.
</Accordion>

<Accordion title="43. LocalStorage vs Cookies vs SessionStorage">
**Answer**:
*   **Cookie**: Sent with Request. Small (4kb). Expiry.
*   **Local**: Persistent. 5MB. JS Access (XSS risk).
*   **Session**: Per tab. Cleared on close.
</Accordion>

<Accordion title="44. JWT Storage">
**Answer**:
**Best**: HttpOnly, Secure, SameSite Cookie. (No JS access = No XSS).
**Common**: LocalStorage (Easier, but risky).
</Accordion>

<Accordion title="45. Shadow DOM">
**Answer**:
Encapsulation.
Styles/JS inside Shadow Root don't leak out.
Web Components.
</Accordion>

<Accordion title="46. PWA (Progressive Web App)">
**Answer**:
Manifest.json (Installable).
Service Worker (Offline).
HTTPS.
App-like experience.
</Accordion>

<Accordion title="47. WebAssembly (Wasm)">
**Answer**:
Binary instruction format. Run C++/Rust in browser.
Near native speed. Video editing, Gaming.
</Accordion>

<Accordion title="48. Single Page App vs Multi Page App">
**Answer**:
*   **SPA**: No reload. JS routing. Complex state.
*   **MPA**: Reloads. Server rendered. Simple.
</Accordion>

<Accordion title="49. Polyfills vs Transpilers">
**Answer**:
*   **Transpiler (Babel)**: Syntax (ES6 -> ES5).
*   **Polyfill**: API (`Promise`, `Array.includes`). Adds missing functionality.
</Accordion>

<Accordion title="50. Micro-Frontends">
**Answer**:
Split large Frontend into independent apps (Team A: Header, Team B: Product).
Module Federation (Webpack).
Complexity overhead.
</Accordion>
</AccordionGroup>
