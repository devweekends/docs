## What is the Linux Kernel Mentorship Program?

The Linux Kernel Mentorship Program (LKMP) helps developers become long-term contributors to the **Linux kernel** — the core of every Android phone, every cloud server, and most of the internet's infrastructure.

This program is run through the **LFX Mentorship platform** but focuses specifically on kernel development. You'll work with experienced kernel developers on real patches that get merged into the mainline Linux kernel.

## Why It's Special

- **You contribute to the Linux kernel** — the most impactful open-source project in history
- **Direct mentorship from kernel maintainers** — people who have been contributing for decades
- **Your patches go into production** — used by billions of devices worldwide
- **Career differentiator** — kernel experience is rare and highly valued by employers (Google, Red Hat, Intel, Meta all hire kernel developers)

## Eligibility

Same as general LFX Mentorship:
- 18+ years old
- Eligible to work in your country
- Not a current maintainer of the subsystem you're applying to
- Basic C programming knowledge and willingness to learn kernel development

## What You'll Work On

Typical kernel mentorship projects include:
- **Bug fixes** — Finding and fixing bugs in kernel subsystems
- **Code cleanup** — Improving code quality, removing deprecated APIs
- **New features** — Implementing functionality in drivers, filesystems, networking
- **Testing** — Writing kernel tests, improving test coverage
- **Documentation** — Improving kernel documentation (yes, it needs help)

Subsystems mentees commonly work on:
- Device drivers
- Filesystem code
- Networking stack
- Memory management
- Security modules

## How to Prepare

Linux kernel development has a steep learning curve. Here's how to prepare:

### Prerequisites
1. **Strong C programming** — The kernel is written almost entirely in C
2. **Linux command line** — Comfortable navigating, compiling, and debugging from terminal
3. **Git proficiency** — Kernel uses git with specific email-based workflow (`git send-email`)
4. **Basic OS concepts** — Processes, memory management, file systems, interrupts

### Recommended Preparation Steps

<Steps>
  <Step title="Set up a kernel development environment">
    Install a Linux distro (Ubuntu/Fedora), clone the kernel source, and learn to build it from source.
  </Step>
  <Step title="Read 'Linux Kernel Development' by Robert Love">
    The standard introductory book for kernel development.
  </Step>
  <Step title="Complete the kernel newbies tutorials">
    [kernelnewbies.org](https://kernelnewbies.org/) has excellent guides for first-time contributors.
  </Step>
  <Step title="Submit your first patch">
    Even a small coding style fix counts. Follow the kernel's patch submission process using `git send-email`.
  </Step>
  <Step title="Join the mailing lists">
    Subscribe to relevant kernel mailing lists for the subsystem you're interested in.
  </Step>
</Steps>

## What Makes a Strong Kernel Mentorship Application

1. **Demonstrate C proficiency with systems-level code** — Show projects that involve memory management, pointer arithmetic, or low-level data structures. User-space C projects (even a simple shell or memory allocator) signal that you can handle kernel code.
2. **Submit at least one kernel patch before applying** — Even a trivial coding style fix through `git send-email` demonstrates that you understand the kernel's unique patch submission workflow. Most applicants never get this far, so even a small patch puts you ahead.
3. **Know the subsystem you are targeting** — Read the MAINTAINERS file for the subsystem, subscribe to its mailing list, and skim recent patch discussions. Mentors want to see that you understand what the subsystem does, not just that you can write C.
4. **Be comfortable with the email-based workflow** — The kernel does not use GitHub PRs. You will use `git format-patch` and `git send-email` to submit patches to mailing lists. Practice this workflow before applying -- it trips up most first-time contributors.
5. **Read "Linux Kernel Development" by Robert Love** — This book is the standard preparation resource. Completing even the first half gives you enough context to have informed conversations with mentors about scheduler internals, memory management, or VFS.
